# Revenue Intelligence Engine

## What & Why
The Revenue Agent described in the OpenClaw system is the most important agent — its job is to continuously track which clients are worth money, which leads are going cold, and where fast revenue is hiding. Right now the app stores all this data but never surfaces any intelligence from it. This task builds the Revenue Agent as a real backend engine with a dedicated intelligence API and client/lead value scoring visible throughout the UI.

## Done looks like
- Every client has a visible value classification badge: High Value (green), Medium Value (amber), or Low Value / Time Waste (red), calculated from total invoiced revenue vs estimated time
- The Leads page shows which leads have gone stale (no update in 7+ days) with a warning indicator and a "Follow Up" prompt
- A `/api/intelligence/revenue` endpoint returns: top clients by revenue, clients with no recent activity, leads past their expected close date, unpaid invoices overdue, and recommended follow-up actions
- A Revenue Intelligence panel appears on the dashboard showing: top 3 revenue opportunities (upsells, inactive clients to reactivate, overdue collections) generated by the engine
- The Clients page shows a sortable "Revenue" column with lifetime value per client
- An "Opportunities" count badge appears on the sidebar nav for Leads and Clients when actionable items are detected

## Out of scope
- Automated email or Stripe integration (the agent advises, it doesn't send emails)
- Per-service pricing analysis
- Forecasting / predictive revenue models

## Tasks
1. **Revenue intelligence API endpoint** — Create `/api/intelligence/revenue` that aggregates: total revenue per client (from paid invoices), leads by staleness (days since last update), overdue invoices, and clients with no activity in 30+ days. Returns structured JSON with `clientRankings`, `staleLeads`, `overdueCollections`, `reactivationTargets`, and `opportunities` array.

2. **Client value scoring** — Compute a value tier (high/medium/low) per client based on total paid invoice revenue thresholds. Add this as a computed field in the clients list API response. Display color-coded value badges on the Clients page and in the client detail view.

3. **Stale lead detection + follow-up indicators** — On the Leads page, show a warning icon and "Stale" label on any lead not updated in 7+ days. Add a sort option "By urgency" that surfaces hot+stale leads at the top.

4. **Revenue opportunities panel on dashboard** — Add a "Revenue Engine" card to the Command Center that calls the `/api/intelligence/revenue` endpoint and displays the top 3 actionable opportunities (e.g. "Reactivate [Client Name] — $X last invoice", "Follow up on [Lead] — 12 days idle", "$X uncollected across N invoices").

5. **Sidebar opportunity badges** — Show a small numeric badge on the Leads and Clients sidebar nav items when the intelligence engine detects actionable items (stale leads count, clients to reactivate).
